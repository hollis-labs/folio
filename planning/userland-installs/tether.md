# Tether Install Readiness Plan

## Current Shape

Tether is the right control plane for the installed suite.

Existing strengths:

- Catalog-driven runtime.
- MCP proxy mode for upstream servers.
- Daemon with Unix socket support.
- HTTP/UDS API.
- Launch profiles, boot profiles, providers, skills, and MCP server catalog
  entries already live in a filesystem catalog.

Current risks:

- Default catalog path is still `~/.agent-mux/catalog`.
- Installed state depends on catalog `global.yaml`, so installer bootstrap must
  be exact.
- Active user config may point at dev repo binaries/DBs unless rendered from an
  installed profile.

## Target Installed Contract

Installed Tether should run with an explicit catalog:

```sh
mux --catalog "$ROOT/tether/catalog" daemon start

mux --catalog "$ROOT/tether/catalog" mcp \
  --proxy \
  --token "$TOKEN" \
  --scopes session.write,message.write
```

Prefer adding aliases/env support later:

```sh
TETHER_CATALOG="$ROOT/tether/catalog" mux mcp --proxy
```

## Required Work

1. Decide rename defaults.
   - Preferred: move installed default from `~/.agent-mux/catalog` to
     `~/Library/Application Support/Hollis Labs/tether/catalog`.
   - Preserve `--catalog` and existing `~/.agent-mux` for dev/backcompat.
2. Add `mux init` or `mux catalog init`.
   - Creates `global.yaml`.
   - Creates subdirectories.
   - Writes state DB and socket paths.
   - Optionally writes baseline provider/agent/launch entries.
3. Ensure `global.yaml` supports app-support paths cleanly.
4. Ensure MCP proxy entries can be fully generated by Folio.
5. Add `mux doctor` or expose enough checks for Folio.
   - Catalog loads.
   - State DB opens.
   - Daemon socket reachable.
   - Upstream MCP discovery works.
6. Add installed catalog fixture tests.
   - Load generated catalog.
   - Start daemon against temp app-support root.
   - Run MCP proxy list-tools against fake/real upstreams.

## Installed Catalog Shape

Folio should render:

```yaml
# global.yaml
defaults:
  state_db: "${ROOT}/tether/tether.db"
  workspace_root: "${ROOT}/tether/workspaces"
daemon:
  listen_addr: "unix:${ROOT}/tether/run/muxd.sock"
  pid_file: "${ROOT}/tether/run/muxd.pid"
```

And:

```text
tether/catalog/
  global.yaml
  mcp-servers/
    nanite.yaml
    torque.yaml
    tesseract.yaml
  projects/
  agents/
  providers/
  launches/
  boot-profiles/
  skills/
```

## Tether MCP Gateway Entry

External MCP clients should need only one entry:

```json
{
  "mcpServers": {
    "tether": {
      "command": "/Users/<user>/Library/Application Support/Hollis Labs/bin/mux",
      "args": [
        "--catalog",
        "/Users/<user>/Library/Application Support/Hollis Labs/tether/catalog",
        "mcp",
        "--proxy"
      ]
    }
  }
}
```

## Acceptance Criteria

- Installed Tether catalog contains no dev repo paths unless explicitly
  configured as project working directories.
- Tether MCP proxy exposes Nanite, Torque, and Tesseract tools.
- Tether daemon can be started/stopped by launchd.
- Folio doctor can prove Tether is using installed state DB and socket paths.
- Existing dev catalog behavior remains compatible.
